# Reporter Agent 지시사항 — Database

## 역할
TDD 사이클 완료 후 PLAN별 결과 보고서를 생성한다.

---

## 보고서 저장 위치

```
Database/reports/PLAN_NN_report.md
```

---

## 보고서 형식

```markdown
# PLAN_NN 결과 보고서

**PLAN**: {번호 및 이름}
**작성일**: {YYYY-MM-DD}
**상태**: PASS / FAIL

## 1. 개발 결과
| 파일 | 위치 | 설명 |
|------|------|------|

## 2. 테스트 결과
| 전체 | PASS | FAIL | 소요 시간 |
|------|------|------|----------|

## 3. 스키마 변경 (해당 시)
| 마이그레이션 파일 | 변경 내용 |
|-----------------|----------|

## 4. 오류 원인 분석
> PASS 시 "해당 없음"

## 5. 리팩토링
| 파일 | 변경 전 | 변경 후 | 이유 |
|------|--------|--------|------|

## 6. 다음 PLAN 권고
```

---

## 수집 정보 출처

| 섹션 | 출처 |
|------|------|
| 개발 결과 | Developer Agent |
| 테스트 결과 | Tester Agent |
| 스키마 변경 | migrations/ 디렉토리 |
| 리팩토링 | Refactor Agent |
